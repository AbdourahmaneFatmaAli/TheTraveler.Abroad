Architectural Overview

The TravelAbroad platform utilizes a traditional Three-Tier Web Architecture to ensure secure and efficient data processing. This structure separates the user interface from the business logic and data storage.

1. Frontend (Client)

Built using Vanilla JavaScript and HTML5. The client handles form validation and sends asynchronous (AJAX) requests to the server, ensuring the page does not need to reload for every action.

2. Backend (Server)

Powered by PHP. The server manages user sessions, hashes passwords for security using password_hash(), and processes booking logic before interacting with the database.

3. Database

A Relational MySQL database stores User profiles, Destination details, and Booking records. All queries use Prepared Statements to prevent SQL injection attacks.
